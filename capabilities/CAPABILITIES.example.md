# Lemonade Capabilities — Example Probe Report

> Example output shape generated by `probe.py`.

| Endpoint | Method | Status | Code | Works | Notes |
|---|---|---|---|---|---|
| `/api/v1/health` | GET | ok | 200 | yes | Lemonade health endpoint reachable |
| `/api/v1/stats` | GET | ok | 200 | yes | Runtime stats available |
| `/api/v1/system-info` | GET | ok | 200 | yes | Host information available |

| `/api/ps` | GET | ok | 200 | yes | Ollama-compatible running-model list available |
| `/api/version` | GET | ok | 200 | yes | Ollama-compatible version endpoint available |

|---|---|---|
| `lemonade config` | yes | Lemonade CLI available |
| `lemonade list --downloaded` | yes | Downloaded model inventory available |
| `lemonade status` | yes | Server status available |
| `systemctl status lemond.service --no-pager` | yes | systemd service visible |
| `journalctl -u lemond.service -n 50 -o cat --no-pager` | yes | journal logs visible |
| `free -h` | yes | Memory information available |
| `sensors` | optional | Depends on host sensor support |
| `/proc/<pid>/cmdline` | optional | Available only when a model process is running |

## Notes

Real probe output is machine-specific. It may include installed model names, process command lines, Lemonade paths, memory totals, OS details, and service logs.

Generated files under `capabilities/results/` and `capabilities/CAPABILITIES.md` are intentionally ignored by git.
